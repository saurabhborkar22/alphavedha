Temporal Split (Never Random!)

Time-series data MUST be split chronologically to avoid look-ahead bias. 20-day embargo gaps prevent data leakage.

Train (70%)
🚫
OOF (15%)
🚫
Val (15%)
Train base models Train ensemble + meta Final evaluation 20-day embargo
Why 3-way split? The ensemble meta-learner needs predictions from base models on data they haven't seen (OOF). If we trained the ensemble on training data predictions, it would overfit to training errors rather than learning genuine model complementarity.
3
Model Training (Dependency Order)

7 models trained in 4 tiers. Each tier depends on the previous tier's outputs.

TIER 1 — BASE LEARNERS (independent, train on Train set)
XGBoost
xgboost.XGBClassifier + XGBRegressor
Input: All 142 features (handles NaN)
Output: Direction (-1/0/+1) + Magnitude
Dual-head: Classifier for direction, Regressor for magnitude
Bonus: Feature importance → selects top 30 for LSTM/TFT
LSTM
PyTorch nn.LSTM (2 layers, 128 hidden)
Input: Top 30 features × 60-day sequences
Output: Direction (-1/0/+1) + Magnitude
Dual-head: 3-class classifier + regression
Why: Captures temporal patterns XGBoost misses
TFT-lite
Custom Temporal Fusion Transformer
Input: Top 30 features × 60-day sequences
Output: 3 horizons: 7d, 15d, 30d
Architecture: VSN → LSTM → Multi-head Attention
Why: Multi-horizon + interpretable attention
Regime Detector
hmmlearn.GaussianHMM (4 states)
Input: Portfolio log returns + realized volatility
Output: P(bull), P(bear), P(sideways), P(high_vol)
States: Bull 🟢 Bear 🔴 Sideways ⚪ High-Vol 🟡
Why: Different models work in different regimes
↓ ↓ ↓ ↓ Base model predictions on OOF set + regime probabilities feed into ensemble
TIER 2 — ENSEMBLE (trained on OOF predictions)
Stacking Ensemble
sklearn.RidgeClassifier (alpha=1.0)
Input: 14 meta-features:
  XGB probas (3) + LSTM probas (3) + TFT probas (3)
  + Regime probs (4) + Disagreement (1)
Output: Final direction + confidence + magnitude
Why: Combines diverse models, each strong in different regimes
↓ ↓ Ensemble predictions + correctness labels feed into meta-labeling
TIER 3 — META-LABELING (trained on ensemble correctness)
Meta-Labeling
xgboost.XGBClassifier (binary:logistic)
Input: Features + ensemble direction + ensemble confidence
Output: P(ensemble prediction is correct) → is_tradeable
Threshold: Only trade when meta_confidence > 0.55
Why: Answers "should I bet on this?" not "which direction?"
Conformal Predictor
MAPIE CrossConformalRegressor (jackknife+)
Input: Features + return magnitudes
Output: [price_low, price_mid, price_high]
Coverage: 90% (intervals expand in high-vol regimes)
Why: Statistically valid prediction intervals
4
What Each Model Learns
Model Question it answers Strength
XGBoost Which features matter most right now? Cross-sectional patterns, feature interactions, handles missing data
LSTM What temporal patterns exist in the last 60 days? Sequential dependencies, momentum shifts, regime transitions
TFT-lite What happens at 7d, 15d, 30d horizons? Multi-horizon, attention-based interpretability, variable selection
Regime HMM Is the market bullish, bearish, sideways, or volatile? Regime-aware predictions, transition probabilities
Ensemble Which model to trust in this market condition? Model diversity, disagreement signal, regime-conditional weighting
Meta-Label Should I even trade this signal? Filters out low-conviction signals, improves win rate
Conformal What's the 90% confidence price range? Statistically valid intervals, wider in volatile markets
